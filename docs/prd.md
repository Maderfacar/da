# 專案需求文件 (PRD)

## 1. 專案概述

本平台提供「即時訂車服務」，讓乘客能透過平台快速預訂機場接送、包車或交通接駁行程，並即時追蹤航班資訊、乘車路線與車程狀態。

**平台核心價值：**
- 統一整合乘客、司機、管理者三方服務流程
- 透過 LINE LIFF 登入實現無縫體驗
- 結合 Google Maps API 與航空 API 實現智慧計價、路線規劃與即時通知
- 幫助乘客安心搭乘、司機高效接單、管理者集中管理

## 2. 目標使用者

1. **訂車乘客**：需要機場接送、包車或交通接駁的旅客（以台灣本島為主）
2. **承接司機**：專業駕駛人員，負責執行訂單任務
3. **平台管理者**：負責訂單管理、司機審核、平台參數設定與媒合調度

## 3. 核心功能

### 乘客端
- 使用 LINE LIFF 登入（自動抓取頭像、名稱，無需額外註冊）
- 個人訂單瀏覽（查看過往訂單歷史）
- 智慧計價與行程規劃：選擇行程項目、車種、上下車地點（含中途停靠站）後，自動計算車資與預估時間（基於 Google API）
- 車種資訊量化顯示（可乘坐人數、行李容量）
- 額外服務選擇

### 司機端
- 司機申請與審核（詳見「§4 司機申請流程」）
- 搶單或瀏覽待接訂單（依 `driverCategory` 排序，預設值為 `'0'`）
- 任務執行頁面（顯示任務詳細內容、出發/到點/上車/下車等回報按鈕）
- 即時位置上傳（最後一次更新位置顯示在地圖上）

### 管理者端
- 採用 **LINE LIFF 登入 + Firestore `users/{uid}.role === 'admin'` 白名單**機制（2026/05/06 統一）
- 新增 / 修改訂單內容
- 指派司機或開放搶單
- 司機管理頁面：分「待審核 / 已核准 / 已拒絕」三分頁，可展開檢視申請資料與證件圖片，可核准 / 拒絕 / 解除冷卻 / 撤銷
- 平台基礎設定（前端版面、文案、計價公式、參數等）
- 搭配航空 API 抓取航班資訊，自動發送即時訊息

### 三端共通 UI
- 三端 Layout Header 右側統一顯示**圓形 LINE 頭像 + displayName**，點擊跳轉至各端 profile 頁
- 若使用者具 admin 權限，於頭像左側顯示「ADMIN」按鈕跳轉至 `/admin/orders`
- 乘客端 Header 簡化：僅保留頭像 + displayName + i18n 語系切換（移除「訂單」「預約」按鈕，與底部 Tab Bar 重複）

## 4. 使用者主要流程

### 乘客流程
1. 透過官方 LINE 頁面進入網站（乘客專屬入口）
2. 點選「訂車」
3. 選擇用車日期與時間
4. 選擇行程類型（送機 / 接機 / 包車 / 交通接送）
5. 設定上下車地點與中途停靠站（支援地理圍欄）
6. 選擇人數、行李數量
7. 選擇車種
8. 選擇額外服務
9. 送出訂單並完成付款（目前僅支援現金支付）

### 司機流程
1. 透過官方 LINE 頁面進入網站（司機專屬入口 `/driver/auth`）
2. LINE LIFF 登入 → 系統依 Firestore `role` 與 `approved` 自動分流：
   - 已核准 driver → `/driver/dashboard`
   - 未核准 driver / 一般 passenger / 新使用者 → `/driver/register`
   - admin → `/admin/orders`
3. 在 `/driver/register` 完成司機申請（詳見「§4 司機申請流程」）
4. Admin 在後台審核：核准 → 司機可進 dashboard；拒絕 → 司機進入 24h 冷卻
5. 選擇「搶單」或瀏覽待接訂單
6. 接受任務後進入任務清單頁面
7. 執行任務時使用按鈕回報狀態（出發 / 到點 / 上車 / 下車），同時上傳位置

## 4. 司機申請流程（New 2026/05/06）

### 4.1 申請表單欄位

**文字欄位（存於 Firestore `users/{uid}.driverApplication`）：**
- 司機真實姓名（與 LINE displayName 分開存：顯示用 LINE 名，後台對帳用真實姓名）
- 聯絡電話
- 車牌號
- 車型：sedan / mpv / suv / van（中英對照顯示，與訂單系統對齊）
- 銀行代號（3 碼）
- 銀行帳號

**圖片欄位（存於 Firebase Storage `drivers/{uid}/`，URL 寫入 Firestore）：**
- 駕照
- 行照
- 保險卡
- 良民證

**其他系統欄位：**
- `driverCategory`：司機分類，預設 `'0'`，admin 審核時可調整為搶單排序權重
- `appliedAt`：申請時間
- `reviewedAt`：審核時間（核准或拒絕當下）
- `reviewedBy`：審核人 uid
- `rejectedAt`：拒絕時間（用於 24h 冷卻判斷；解除冷卻時清空）
- `rejectReason`：拒絕原因（admin 填寫）

### 4.2 申請狀態與重申冷卻

| 狀態 | 觸發條件 | `/driver/register` 顯示 |
|------|---------|----------------------|
| 申請表單 | `role=passenger` 或新使用者 | 完整表單 |
| 審核中 | `role=driver, approved=false`, 無 `rejectedAt`（或已過 24h） | 「審核中」提示 + 申請內容 |
| 冷卻中 | `rejectedAt` 在 24h 內 | 倒數計時 + 拒絕原因 |
| 已核准 | `role=driver, approved=true` | 自動導向 `/driver/dashboard` |

**冷卻機制**：被拒絕 24 小時內 `apply` API 拒絕新提交（回傳 `403`）；admin 可在 `/admin/drivers` 點「解除冷卻」立即清空 `rejectedAt`。

## 5. 範圍限制

- 禁止選擇台灣本島以外的地點（包含離島及海外）
- 目前不支援信用卡、電子支付等非現金付款方式
- 不提供即時叫車（僅預約制）
- 不開發獨立的 App（以 Web + LINE LIFF 為主）
- 不處理跨國或國際航班接送業務

## 6. 設計風格

**設計主題：美式復古機場風（Retro Airport Americana）**

- Bebas Neue 粗體大標、黃黑斜條紋分隔線（`stripe-divider`）
- 機場代碼浮水印（TPE / JFK / NRT）— Ghost text 絕對定位，opacity 6%
- 主色調：米白 `#F5F2EC` + 暖奶油 `#FAF8F4`，琥珀金點綴 `#D4860A`
- 毛邊玻璃效果（`backdrop-filter blur`）用於表單卡片、Trip Card、導航列
- 飛機動畫、浮動效果（`floatY`）
- 統計數字展示採**機場翻牌效果**（Split-Flap Board）取代跑馬燈
- 行動版優先（Mobile First），所有頁面優先設計 375px 手機版面
- 完整規範見 [docs/style-guide.md](style-guide.md)

## 7. 其他重要需求

- **多語系**：預設繁體中文（zh），同時支援英文（en）與日文（ja）
- **LINE 整合**：使用 LINE LIFF 登入 + LINE Bot 推送通知
- **支付方式**：目前僅支援現金支付
- **地理圍欄**：上下車地點及中途停靠站必須限制在台灣本島範圍內
- **航空 API**：需整合航班資訊 API（具體廠商尚未確定）
- **行動優先**：所有頁面必須在手機上擁有良好使用體驗
